# Project Structure

song2quiz/
├── src/               # Source code
│   ├── flows/         # Prefect workflows
│   ├── services/      # External API integrations
│   ├── tasks/         # Individual task definitions
│   └── utils/         # Utility functions
├── tests/             # Test suite
├── docs/              # Documentation
└── data/              # Data storage

## Key Components

### Source Code (`src/`)
The `src/` directory contains all the application logic, organized into several key modules:

- **flows/**: Prefect workflow definitions for orchestrating tasks
- **models/**: Data models and type definitions
- **prompts/**: Templates for LLM interactions
- **services/**: External API integrations
- **tasks/**: Core business logic implementations
- **utils/**: Helper functions and utilities

### Data Storage (`data/`)
- **songs.json**: Central index of all processed songs
- **songs/**: Individual song data files

### Supporting Directories
- **tests/**: Test suite mirroring the src/ structure
- **docs/**: Project documentation
- **scripts/**: Development and maintenance scripts
- **resources/**: Static assets and resources

## Key Files

- `requirements.txt`: Python dependencies
- `data/songs.json`: Central song metadata index
- `.env`: Environment configuration (API keys, etc.)

## Development Guidelines

1. New features should be implemented as Prefect flows in `src/flows/`
2. Data models should be defined in `src/models/`
3. Core business logic belongs in `src/tasks/`
4. Tests should mirror the `src/` directory structure

## Common Operations

### Adding a New Feature
1. Define models in `src/models/`
2. Implement core logic in `src/tasks/`
3. Create Prefect flow in `src/flows/`
4. Add tests in `tests/`

### Processing New Songs
1. Song data is ingested via `src/flows/ingestion/`
2. Metadata is stored in `data/songs.json`
3. Individual song data goes to `data/songs/`
